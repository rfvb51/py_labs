# Лабораторная работа №5: Функции как аргументы. Стратегии и делегаты.

## 1. Цель работы

Изучение передачи функций как аргументов в другие функции и методы. Освоение встроенных функций высшего порядка `map`, `filter`, `sorted`. Понимание концепции паттерна «Стратегия» и его реализация на Python. Применение `lambda`-выражений. Интеграция функционального стиля с объектно-ориентированным кодом.

## 2. Реализованные функции и стратегии

### Функции-стратегии сортировки (3 шт.)

- `by_route_number(bus)` — сортировка по номеру маршрута
- `by_passenger_count(bus)` — сортировка по количеству пассажиров
- `by_load_factor(bus)` — сортировка по загрузке в процентах

### Функции-фильтры (2 шт.)

- `has_passengers(bus)` — отбирает автобусы с пассажирами
- `is_tourist_bus(bus)` — отбирает только туристические автобусы

### Фабрика функций

- `make_capacity_filter(min_capacity)` — создаёт фильтр по минимальной вместимости. Возвращает функцию, которую можно использовать в `filter_by()`.

### Callable-объект (паттерн Стратегия)

- `DiscountLoadStrategy` — класс с методом `__call__()`. Уменьшает скорость автобуса на 20%, если загрузка ниже 30%.

### Методы коллекции `AdvancedBusCollection`

- `sort_by(key_func)` — возвращает новую коллекцию, отсортированную по ключу
- `filter_by(predicate)` — возвращает новую коллекцию с отфильтрованными элементами
- `apply(func)` — возвращает новую коллекцию с применённой функцией к каждому элементу

## 4. Вывод

В ходе лабораторной работы было изучено:

**Передача функций как аргументов**
- Функции могут быть переданы в другие функции и методы (`sort_by(by_route_number)`)

**Функции высшего порядка**
- `map()` — преобразование коллекции
- `filter()` — фильтрация коллекции
- `sorted()` — сортировка с key-функцией

**Lambda-выражения**
- Создание анонимных функций для простых операций
- Сравнение lambda и именованных функций

**Замыкания и фабрики функций**
- Создание функций, возвращающих другие функции с захваченными параметрами

**Паттерн «Стратегия»**
- Вынесение алгоритмов в отдельные функции/классы
- Взаимозаменяемость стратегий без изменения кода коллекции
- Callable-объекты как альтернатива функциям

**Интеграция с ООП**
- Методы `sort_by()`, `filter_by()`, `apply()` в классе коллекции
- Цепочки операций для функционального стиля
