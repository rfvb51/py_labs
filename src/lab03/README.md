## 2. Описание реализованной иерархии классов

### Базовый класс `Bus`

**Атрибуты:**
- `_route_number` — номер маршрута
- `_capacity` — вместимость (1-50)
- `_passenger_count` — количество пассажиров
- `_current_speed` — текущая скорость (0-120)
- `_state` — состояние (in_depot / on_route / maintenance)

**Методы:**
- `start_route()` — выезд на маршрут
- `return_to_depot()` — возврат в депо
- `send_to_maintenance()` — отправка на ТО
- `repair_complete()` — завершение ТО
- `board_passengers(count)` — посадка пассажиров
- `alight_passengers(count)` — высадка пассажиров
- `free_seats()` — свободные места
- `load_factor()` — загрузка в процентах
- `calculate_fare()` — расчет стоимости проезда

### Дочерний класс `TouristBus`

**Добавленные атрибуты:**
- `_has_wifi` — наличие Wi-Fi
- `_has_ac` — наличие кондиционера
- `_route_type` — тип маршрута (фиксирован "междугородний")

**Добавленные методы:**
- `display_comfort()` — информация о комфорте

**Переопределенные методы:**
- `calculate_fare()` — базовая стоимость × множитель (1.5 + 0.2 за Wi-Fi + 0.3 за кондиционер)
- `__str__()` — добавлена информация о Wi-Fi и кондиционере
- `__repr__()` — добавлены новые параметры

### Дочерний класс `SchoolBus`

**Добавленные атрибуты:**
- `_route_type` — тип маршрута (городской / пригородный / междугородний)
- `_has_supervisor` — наличие сопровождающего

**Добавленные методы:**
- `get_safety_info()` — информация о безопасности
- `can_transport_children()` — может ли перевозить детей

**Переопределенные методы:**
- `calculate_fare()` — всегда возвращает 0 (бесплатный проезд)
- `__str__()` — добавлена информация о типе маршрута и сопровождающем
- `__repr__()` — добавлены новые параметры
