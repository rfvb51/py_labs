# Лабораторная работа №6: Generics и typing

## 1. Цель работы

Освоение системы аннотаций типов в Python, создание обобщённых (generic) классов с помощью `TypeVar` и `Generic`, понимание структурной типизации через `typing.Protocol`.

## 2. Описание реализованных типов и контейнеров

**Generic-класс `TypedCollection[T]`**
- Обобщённая коллекция, работающая с любым типом элементов
- Тип указывается при создании: `TypedCollection[Bus]()`
- Содержит методы: `add`, `remove`, `get_all`, `find`, `filter`, `map`, `sort_by`, `__len__`, `__getitem__`, `__iter__`

**TypeVar**
- `T` — основной тип элементов коллекции
- `R` — тип результата в методе `map` (позволяет менять тип)
- `D` — с ограничением `bound=Displayable` (только объекты с методом `display()`)
- `S` — с ограничением `bound=Scorable` (только объекты с методом `score()`)

**Protocol (структурная типизация)**
- `Displayable` — требует метод `display() -> str`
- `Scorable` — требует метод `score() -> float`
- Классы не наследуют протоколы явно — достаточно наличия методов

## 4. Вывод

В ходе лабораторной работы было изучено:

**Аннотации типов**
- Указывают ожидаемые типы параметров и возвращаемых значений
- Помогают IDE и статическим анализаторам находить ошибки

**Generic и TypeVar**
- Позволяют создавать классы, работающие с разными типами
- `TypedCollection[T]` — одна реализация для любых типов
- Второй TypeVar `R` в `map()` показывает смену типа результата

**Protocol (структурная типизация)**
- Проверяет наличие методов, а не наследование
- «Утиная типизация» с проверкой на этапе анализа кода
- TypeVar с `bound=Protocol` ограничивает коллекцию только подходящими объектами

**Преимущества типизации**
- Код становится самодокументируемым
- Ошибки обнаруживаются раньше
- Упрощается работа в команде и поддержка кода
