# Лабораторная работа №4: Интерфейсы и абстрактные классы (ABC)

## 1. Цель работы
Изучение абстрактных базовых классов (ABC) и интерфейсов в Python. Освоение создания контрактов поведения, обязательных для реализации в дочерних классах. Закрепление полиморфизма через единый интерфейс.

## 2. Описание интерфейсов

Создано два абстрактных класса (интерфейса) в файле `interfaces.py`:

**Printable**
- `to_string()` — возвращает строковое представление объекта

**Comparable**
- `compare_to(other)` — сравнивает текущий объект с другим, возвращает разницу

Оба интерфейса наследуют от `ABC` и используют декоратор `@abstractmethod`.

## 3. Реализация в классах

**CityBus** (городской автобус)
- Реализует: Printable, Comparable
- `to_string()`: возвращает маршрут, количество пассажиров и доход
- `compare_to()`: сравнивает по доходу

**TouristBus** (туристический автобус)
- Реализует: Printable, Comparable
- `to_string()`: возвращает имя гида, пассажиров и доход
- `compare_to()`: сравнивает по доходу

**SchoolBus** (школьный автобус)
- Реализует: только Printable
- `to_string()`: возвращает школу, наличие сопровождающего и пассажиров
- `compare_to()`: не реализован (школьные автобусы не сравниваются)

**Отличия в поведении:**
- `to_string()` у каждого класса возвращает разную информацию (маршрут и доход / гид и доход / школа и сопровождающий)
- `calculate_fare()` вызывается внутри `to_string()` и также ведет себя по-разному (платный / платный с наценкой / бесплатный)
- SchoolBus не реализует Comparable, в отличие от двух других
