# Лабораторная работа №1: Класс и инкапсуляция

**Свойства (@property)** — механизм, позволяющий определять методы, которые можно использовать как атрибуты, с контролем доступа и валидацией.

## Вариант 5: Транспорт

**Сущности:** Car, Bus, Ticket, Route, Driver

**Реализовано:** класс Bus (автобус)

### Закрытые поля:
- `_route_number` — номер маршрута
- `_capacity` — вместимость автобуса (макс. 50)
- `_passenger_count` — текущее количество пассажиров
- `_current_speed` — текущая скорость
- `_state` — состояние автобуса (in_depot/on_route/maintenance)

### Свойства @property:
**Чтение:**
- `route_number` — номер маршрута
- `capacity` — вместимость
- `passenger_count` — количество пассажиров
- `state` — состояние автобуса

**Чтение и запись:**
- `current_speed` — скорость (с валидацией и проверкой состояния)

### Магические методы:
- `__str__` — пользовательское описание (для print)
- `__repr__` — представление для разработчика (можно воссоздать объект)
- `__eq__` — сравнение объектов по состоянию

### Бизнес-методы:
**Управление состоянием:**
- `start_route()` — выезд на маршрут
- `return_to_depot()` — возврат в депо
- `send_to_maintenance()` — отправка на ТО
- `repair_complete()` — завершение ТО

**Работа с пассажирами:**
- `board_passengers(count)` — посадка пассажиров (возвращает количество севших)
- `alight_passengers(count)` — высадка пассажиров (возвращает количество высадившихся)

**Расчетные методы:**
- `free_seats()` — количество свободных мест
- `load_factor()` — коэффициент загрузки в процентах
